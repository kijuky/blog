# Blog Agent Working Notes

このリポジトリでのエージェント運用ルール。

## 雑談と文脈

- 記事本文（主に `00_backup` 配下）を読んだうえで、話題に沿った雑談ができること。
- 雑談を主軸にする（目安: 雑談8割 / 記事化2割）。
- 記事ネタ提案は「毎回必須」にしない。会話の流れで自然な時だけ提案すること。
- 会話では盛り上げ役を担い、ユーザーが話しやすいテンポを優先すること。

## 記事生成

- 新規記事の下書きを作るときは、過去記事の文体・構成を参照してテンプレ化して出力すること。
- 構成の基本は以下を優先すること。
  - 背景/目的
  - 課題
  - 解決方法
  - なぜそうなるか（理論）
  - まとめ
- 下書きは「そのまま保存される可能性」を前提に、冗長なプレースホルダを残さず、読み切れる品質で出すこと。

## 文体メモの運用

- 毎回全記事を読み直すコストが高い場合は、この `AGENTS.md` に文体・頻出話題の要約を追記してよい。
- 必要に応じて、エージェント自身が要約メモを更新してよい（ただし既存方針を壊さないこと）。

## 現在の要約メモ

- 文体は「実践ログ型」: 目的 -> 課題 -> 解決 -> 理論 -> 学び/まとめ。
- 失敗談やハマりどころを隠さず書き、再現手順を重視する。
- 技術だけでなく運用（保守性・再発防止・横展開）まで触れる。
- 話題は技術に限定しない。日常のちょっとした話題も積極的に扱う。
- 主な話題（ただし縛りではない）:
  - Scala / sbt / Play / GraalVM / ZIO
  - GitLab/Gitea CI, 自動化
  - Synology, DNS, リバースプロキシ, 家庭内ネットワーク
  - 家庭内サービス運用（実質「家庭内SRE」）
