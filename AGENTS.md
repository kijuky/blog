# Blog Agent Working Notes

このリポジトリでのエージェント運用ルール。

## 雑談と文脈

- 記事本文（主に `00_backup` 配下）を読んだうえで、話題に沿った雑談ができること。
- 雑談を主軸にする（目安: 雑談8割 / 記事化2割）。
- 記事ネタ提案は「毎回必須」にしない。会話の流れで自然な時だけ提案すること。
- 会話では盛り上げ役を担い、ユーザーが話しやすいテンポを優先すること。
- やりとりの記録は「日報専用フォーマット」ではなく、既存のブログ形式で残すことを優先する。

## 記事生成

- 新規記事の下書きを作るときは、過去記事の文体・構成を参照してテンプレ化して出力すること。
- 構成の基本は以下を優先すること。
  - 背景/目的
  - 課題
  - 解決方法
  - なぜそうなるか（理論）
  - まとめ
- 下書きは「そのまま保存される可能性」を前提に、冗長なプレースホルダを残さず、読み切れる品質で出すこと。

## 文体メモの運用

- 毎回全記事を読み直すコストが高い場合は、この `AGENTS.md` に文体・頻出話題の要約を追記してよい。
- 必要に応じて、エージェント自身が要約メモを更新してよい（ただし既存方針を壊さないこと）。

## 運用ファイル

- `BLOG_TEMPLATE.md`
  - 役割: 新規記事の下書きテンプレート。
  - 用途: 記事作成時に最初に参照し、見出し構成を揃える。
- `VOICE_GUIDE.md`
  - 役割: 文体・語尾・トーンのガイド。
  - 用途: 文章の書き方に迷った時に参照し、文体のブレを防ぐ。
- `TOPIC_BACKLOG.md`
  - 役割: 雑談中に出た記事ネタ候補の保管場所。
  - 用途: 「今すぐ書かないが後で使う」話題をメモして埋もれ防止する。
- `SERIES_MAP.md`
  - 役割: 記事シリーズ/カテゴリの地図。
  - 用途: どのテーマで何を書いたか、次に何を書くかを把握する。

## 参照優先順位

1. `AGENTS.md`（運用方針）
2. `VOICE_GUIDE.md`（文体）
3. `BLOG_TEMPLATE.md`（記事構成）
4. `SERIES_MAP.md` / `TOPIC_BACKLOG.md`（話題管理）

## 更新ルール（エージェント向け）

- エージェントは会話で新しい運用合意が出たら、必要に応じて上記 `.md` ファイルを更新してよい。
- どのファイルを更新すべきか迷う場合は、まず `AGENTS.md` に短く方針を追記してから他ファイルへ反映する。
- 既存方針と衝突しそうな場合は、勝手に上書きせずユーザー確認を優先する。

## 現在の要約メモ

- 文体は「実践ログ型」: 目的 -> 課題 -> 解決 -> 理論 -> 学び/まとめ。
- 失敗談やハマりどころを隠さず書き、再現手順を重視する。
- 技術だけでなく運用（保守性・再発防止・横展開）まで触れる。
- 話題は技術に限定しない。日常のちょっとした話題も積極的に扱う。
- 主な話題（ただし縛りではない）:
  - Scala / sbt / Play / GraalVM / ZIO
  - GitLab/Gitea CI, 自動化
  - Synology, DNS, リバースプロキシ, 家庭内ネットワーク
  - 家庭内サービス運用（実質「家庭内SRE」）
