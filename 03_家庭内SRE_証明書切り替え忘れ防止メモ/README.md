# 家庭内SRE: 証明書切り替え忘れ防止メモ

## 背景

家庭内サービスを Docker で増やしていくと、最初は「DNSで名前解決できればOK」と思いがちだった。  
ただ実際は、IPだけでなく `IP + ポート` を名前で扱いたくなる。

その結果、うちの構成は次の形になった。

- DNS: 名前をSynologyのIPに向ける
- リバースプロキシ: Host/Pathで実サービス（コンテナのポート）へ振り分ける

しかも DNS もリバースプロキシも Synology (DSM) 上で完結するため、どちらを触っていたか混線しやすい。

## 起きたこと

新しいルートを追加した時に、リバースプロキシ側の証明書割当を切り替え忘れた。  
結果、HTTPS警告で無事に死んだ。

## 原因

責務分離は理解していたが、運用手順に落とし込めていなかった。

- DNS設定は更新した
- リバースプロキシ設定も更新した
- 証明書割当の最終確認だけ漏れた

## 最小運用ルール（再発防止）

変更時は以下だけ必ず確認する。

1. DNS
   - `A/AAAA` が Synology を向いているか
2. リバースプロキシ
   - Host/Path ルールが正しいか
3. 証明書
   - 対象ルートに正しい証明書が割り当たっているか

## 1行チェックリスト

新規サービス追加時は、作業ログにこの1行をコピペして使う。

`[ ] DNS  [ ] Reverse Proxy  [ ] Certificate`

## まとめ

家庭内インフラでも、障害の多くは設定漏れで起きる。  
特に「同じ画面で複数責務を触れる環境」は漏れやすいので、運用ルールを短いチェックリストに固定するのが効く。
