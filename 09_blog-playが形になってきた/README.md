blog-playがどんどん形になってきた。ogp対応するとかなり見栄えが良くなって、自分の文章を見るのはこのビューワーでいいな、って気になってくる。sbt runで起動していればブログ内容が変わればホットリロードされるのも良い。起動時に全記事DBに入れ直しているので瞬時にと言うわけには行かないが、今のところ待てない時間ではない。ZIO版を起動するとこの初回動作が驚くほど速いのでScalikeJDBC vs Quillの違いなのかな？と思っている。

ogp取得が並列すぎるのか、たまにtoo many accessになって画像等が取得できないことがあるので、ogpの同時取得数は制限した方が良いのかもしれない。

あと、引用のレイアウトが簡素すぎるので、そこも変えたい。

code highlightの亜種としてprocessing.jsに対応できれば、もっと面白い表現ができるかもしれないなと思った。昔作ったゲームをJSに移植したかったが、Processingの方が自分にとって親しみがあるかもしれない。

今、フロントはtypescriptで書いているが、sbt webの仕組みだと単体テストが書きにくいので、scalajsに書き換えたい。またそうすればフロント資産はplay/zioで共有できそうだ。

そうなると、このリポジトリもblog-playというよりかは単にblog-viewerとかにした方が良さそうだな...もともとはblog-playとblog-zioを用意するつもりだったけど、一つのリポジトリで事足りてしまった。
