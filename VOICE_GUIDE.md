# Voice Guide

このリポジトリで記事を書くときの文体ガイド。

## 基本トーン

- 実践的・具体的・再現性重視。
- 失敗やハマりどころを隠さない。
- 読者が次に試せる情報を残す。

## 構成の癖

- 「背景 -> 課題 -> 解決 -> 理論 -> まとめ」を基本にする。
- 先に結論、その後に経緯を書く。
- 必要なら箇条書きを優先して可読性を上げる。

## 言い回し

- 説明は短文寄りで、曖昧語を減らす。
- 断定しすぎない（前提条件や例外を添える）。
- ただし冗長にはしない。

## NG

- 中身のない一般論だけで終わる。
- 事実確認なしの断定。
- プレースホルダだらけの下書きを出す。
