自分はSlack通知botをよくGASで作っているのですが、GASだと社内オンプレサービスに接続できない事が多いと思います。今回、社内オンプレにつながるGitLabを使ってSlack通知できたので、それをまとめたいと思います。

# GitLab のパイプラインスケジュールを活用

GitLab には[パイプラインスケジュール](https://docs.gitlab.com/ee/ci/pipelines/schedules.html)という機能があります。これを使って特定日時に社内サービス(RedmineとかJIRAとか)にアクセスして、そのレポートをSlackに通知します。

## 環境変数の設定

このパイプラインスケジュールにはスケジュールごとに環境変数を設定できます。例えばSlack のWebhookが設定されている場合だけSlackに通知するようにして、それ以外は標準出力に出力するようにしておけば、デバッグ時は標準出力で確認できて、スケジュール起動のときはSlackに通知するようにできます。
