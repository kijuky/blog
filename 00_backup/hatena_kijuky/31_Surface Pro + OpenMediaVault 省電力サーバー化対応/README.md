## やろうとしたこと / 目的
- Surface Pro 初代に導入した OpenMediaVault を無線LAN接続で常時稼働させたい
- 有線接続を外し、設置自由度を上げる
- 消費電力を抑えるため画面は消灯させたい
- NAS用途として安定運用できる状態にする

## 発生した課題・問題
1. 無線LANが認識されない（過去は接続できていた）
2. 画面消灯設定を入れると Wi-Fi も停止して OMV に接続不能になる
3. Linuxのログイン情報を失念し、設定変更ができない
4. nano エディタの操作方法が分からない
5. 文字化けにより設定ファイル編集が困難

## 解決策・対応内容

### 1. rootログインの確保
- リカバリ的手順で root にログイン成功
- これにより設定ファイル編集が可能になった

### 2. サスペンド無効化
- 画面消灯とスリープ（サスペンド）が別物であることを確認
- サスペンドに入ると無線LANデバイスの電源が落ちるため通信断が発生していた

対応：
- systemd の電源管理設定を変更
- 自動サスペンドを無効化

### 3. 画面のみ消灯させる設定
- コンソールのブランク設定を使用
- 画面表示だけ停止し、OSは稼働を継続させた

### 4. Wi-Fi維持の確認
- サスペンドに入らなくなったことで、無線LANインターフェースの電源断が解消
- OMVへ無線接続で継続アクセス可能になった

## なぜ解決できたか（理論）

Linuxには以下の電源状態がある：

| 状態 | 内容 |
|---|---|
| 画面消灯（console blank） | 表示のみ停止、OSは動作継続 |
| idle | CPU低電力化、通信維持 |
| suspend (mem) | RAMのみ保持、デバイス電源断 |
| hibernate | メモリ内容をディスク保存して停止 |

今回の問題は「画面消灯」ではなく  
**自動サスペンド（suspend to RAM）に入っていたこと** が原因。

サスペンド時は
- PCIe
- USB
- Wi-Fiチップ

の電源が切れるため、ネットワークが消失し NAS が停止したように見えていた。

サスペンドを禁止し、画面消灯のみ有効にしたことで：

- OSは稼働
- ネットワーク維持
- 消費電力低減

が同時に成立した。

## 現在の状態
- 無線LAN接続で OMV 稼働可能
- 画面消灯中もネットワーク維持
- 常時稼働NASとして利用可能
- 低消費電力運用を実現
