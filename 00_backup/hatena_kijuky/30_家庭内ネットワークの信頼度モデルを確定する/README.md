## やろうとしたこと
- 旧PCを家庭内LANに安全に参加させる構成を決める
- VLANを導入せず、現行機材でどこまで安全性を確保できるか整理する
- SSID構成と役割分担を明確化する
- パスワード運用の妥当性を検討する

## 発生した課題
- 旧PCをLANに入れた場合の横移動（ラテラルムーブメント）リスク
- VLANを導入するかどうか未決定（＝選択が必要な状態）
- UDR7購入というコスト判断が未確定
- SSIDごとの役割分離が言語化されていない
- パスワードを共通にするか分離するか未決定
- 旧端末・IoT機器の扱い方針が曖昧

※いずれも不具合ではなく、「選択が必要な状態」を課題として扱う

## 解決策（決定事項）
- VLANは導入しない（現状機材で成立させる方針）
- 信頼度レベルでネットワークを分離する思想を採用
    - メイン系
    - ゲスト系
    - 旧端末／制限系
- 旧PCとのファイル共有はNAS経由に限定する
- NASの構成を中継点として活用する設計を許容
- SSIDは役割ごとに明確に分離し維持する
- 2.4GHz帯は20MHz固定で安定性優先

## 理論・背景
- 横移動対策としてネットワーク境界を作る
- VLANがなくても、論理的な信頼度分離でリスクは低減できる
- 帯域幅拡張は再送増加リスクを伴うため、2.4GHzは安定重視とする
- 家庭内LANも「単一ネットワーク」ではなく、信頼度の異なるセグメント集合として扱うのが妥当
