# やりたかったこと

- 家庭内ネットワークの固定IP割り当てを整理し、機器ごとの役割が一目でわかるようにしたい。
- 特にサーバー系、周辺機器、メディア機器、ゲーム機の固定IPを管理しやすくすることが目的。

# 発生した課題

- 機器が増えるたびにIPアドレスの管理が煩雑になり、どれが固定でどれがDHCPか判別しづらい。
- 固定IPを手動で設定すると重複やトラブルのリスクがある。
- ゲーム機やメディア機器はIPが変わるとサービスや接続が不安定になることがある。

# 課題の解決方法

1. IPを役割ごとにゾーン分けして番号を振る

   | 範囲 | 用途 | 割り当て機器(例) |
       | :-: | :- | :- |
   | .1 | ゲートウェイ | ルーター |
   | .2–.19 | ネットワーク機器 | メッシュ中継機、スイッチ、APなど |
   | .20–.39 | サーバー/ストレージ | 仮想化サーバー、NASなど |
   | .40–.59 | PC周辺機器 | 複合機、スキャナなど
   | .60–.79 | AV・メディア機器 | テレビ、DVD/BDレコーダーなど |
   | .80–.89 | ゲーム機 | Nintendo Switch(有線) |
   | .90–.99 | 予備/IoT | スマート家電・カメラなど |
   |.100–254 | DHCP | スマホ・PC・タブレットなど自動取得 |
2. 固定IPはルーターのDHCP予約で一元管理
    - 機器側で手動設定せず、MACアドレス単位で固定
3. 将来増えるIoTやスマート家電用に予備ゾーンを確保
    - 90〜99番など

# 解決方法の理論的説明

- ゾーン分けにより、IPを見ただけで機器のカテゴリが分かるためトラブル対応が容易。
- DHCP予約管理はIP重複のリスクを回避し、ルーター交換時でも設定を一元復元可能。
- 将来用予備ゾーンをあらかじめ確保することで、新規機器追加時も既存構成を乱さず運用可能。
- この整理により、家庭内のネットワークの安定性・可視性・拡張性が向上する。
