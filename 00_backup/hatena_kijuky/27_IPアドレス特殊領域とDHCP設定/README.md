# やりたかったこと

- 家庭内LANで使用するDHCPの範囲設定を安全にしたい
- 特殊なIPアドレス（127.0.0.1、192.168.0.0/24のネットワーク・ブロードキャストなど）を把握し、誤って端末に割り当てないようにしたい

# 発生した課題

- 特殊アドレス（ネットワークアドレス・ブロードキャストアドレス）をDHCPで誤配布すると通信が不安定になる
- 127.0.0.1などのループバックアドレスやAPIPA、マルチキャストアドレスも誤配布するとトラブルの原因になる

# 課題の解決方法

- DHCPで配布する範囲を明確に分ける
- ネットワークアドレス（例：192.168.0.0）とブロードキャストアドレス（例：192.168.0.255）は除外
- ルーターやサーバなど固定機器用のIPは予約
- 一般端末には安全な範囲（例：192.168.0.100〜199）をDHCPで配布

# 解決方法の理論的説明

- ネットワークアドレスはそのネットワーク自体を表す識別子であり、端末に割り当てると通信路が不明瞭になる
- ブロードキャストアドレスはネットワーク全体宛の通信に使われるため、端末に割り当てると全体通信やARP、DHCP Discoverが正常に機能しなくなる
- ループバックアドレスは自分自身向け通信専用であり、DHCPで配布すると内部通信が壊れる
- DHCPで安全に配布することで、これら特殊アドレスと端末のIPが競合せず、ネットワークの安定性が保たれる
