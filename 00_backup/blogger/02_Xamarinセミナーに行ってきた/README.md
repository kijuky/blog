Xamarin  
iOS/Android/MacのネイティブのアプリをC#で書ける  
バックエンドはXamarinの共通コード、その上にOS毎のコードが乗る  
C#かける→非同期サポートasync.await, LINQ, ラムダ式が書ける  
System系が動く。WindowsやiOSに特徴的なusingが使える  
XamarinでiOSを作る場合はWindows(Visual Studio)とMac（実機がシミュレータを接続するため）が必要  
XamarinでAndroidを作る場合はVisualStudioだけでよい  
メモ：受け入れテスト　
XamarinでMacのアプリを開発する場合はMacだけで作る  
Xamarinを使うことでビジネスロジックを共有化できる（個別のデバイスのUIの差だけに注力できる）  
共通部分はNuGetでとってこれる！楽ちん！  
Xamarin.Forms→UIコードも共通化、XAMLやC#で書く。ビルドするとネイティブのUIにリンクされる  
Xamarinで書いたとしても各デバイスの開発知識は必要  
Xamarinは高い！
